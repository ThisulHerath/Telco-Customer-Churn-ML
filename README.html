// quick start

Get Running in 60 Seconds

▸ pull & run from docker hub
# Pull the latest image
docker pull 00pizzalover00/churn-service:latest

# Run on port 8000
docker run --rm -p 8000:8000 00pizzalover00/churn-service:latest
▸ local development
# Create virtual environment
python -m venv .venv && source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Run tests
pytest -q

# Start the API (interactive docs → http://localhost:8000/docs)
uvicorn src.app.main:app --reload --host 0.0.0.0 --port 8000
▸ build locally with docker
docker build -t local-churn-service:dev .
docker run --rm -p 8000:8000 local-churn-service:dev
// ci/cd pipeline

What Happens on Push

📥

Checkout

Repository code is checked out in a fresh runner.

🐍

Setup Python

Environment created and requirements.txt dependencies installed.

🧪

Run Tests

pytest runs unit and integration tests — failure blocks merge.

🔍

Lint & Static Checks

Optional linting stage validates code quality.

🐳

Docker Build

Image built and tagged (latest + commit/semver tag).

🔐

Authenticate

Docker Hub login via repo secrets DOCKER_USERNAME / DOCKERHUB_TOKEN.

🚢

Push to Docker Hub

Image published to 00pizzalover00/churn-service.

📊

Artifact Upload

Optionally saves model files to MLflow, remote storage, or GitHub Releases.

Workflow file: .github/workflows/ci.yml
// model artifacts

MLflow & Feature Alignment

📁 mlruns/

Tracked runs and saved models for local MLflow. Set MLFLOW_TRACKING_URI for remote tracking.

📋 feature_columns.json

Stored under artifacts/ and src/serving/model/ — aligns new requests with the model's expected input schema.

🔬 Serving

FastAPI app reads the serialised model + feature columns JSON. See src/serving/inference.py.

☁️ Remote

Set MLFLOW_TRACKING_URI and MLFLOW_S3_BUCKET for cloud artifact storage in CI.

How to Contribute

# Create a feature branch
git checkout -b feature/your-feature

# Run tests before opening a PR
pytest tests/ -q

🌿 Branch

One branch per feature or bugfix. Keep PRs focused and reviewable.

✅ Tests

Write tests for new functionality. CI blocks merges on test failures.

🔒 Secrets

Never commit credentials. Use GitHub Actions secrets or your cloud secret manager.
